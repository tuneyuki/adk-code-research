# ADK 実行モードとサーバースタックの分析

本ドキュメントでは、ADK (Agent Development Kit) で利用可能な各実行モードについて詳述します。特に、基盤となる技術スタック、デーモンとしての実行への適合性、および Server-Sent Events (SSE) とクライアント切断時の挙動に焦点を当てています。

## 1. デーモン / APIサーバーモード (`adk api_server`)

これは、ADK をバックグラウンドサービス（デーモン）として実行し、SSE ストリームを含む API リクエストを受け付けるために推奨されるコマンドです。

- **コマンド**: `adk api_server [AGENTS_DIR] [OPTIONS]`
- **主なユースケース**: 本番環境やヘッドレス環境でのエージェント API の提供。
- **技術スタック**:
    - **ASGI サーバー**: **Uvicorn**
    - **フレームワーク**: **FastAPI** (Starlette)
    - **エンドポイント**: `/run_sse` (ストリーミングリクエストを受け付け可能)

### デーモン実行時の主要オプション

| オプション | デフォルト | 説明 |
| :--- | :--- | :--- |
| `--host` | `127.0.0.1` | バインドアドレス。外部からのアクセスを許可するには `0.0.0.0` を使用します。 |
| `--port` | `8000` | リッスンするポート番号。 |
| `--no-reload` | (デフォルトで有効) | **本番運用で推奨。** ファイル変更を監視する自動リローダーを無効にします。 |
| `--session_service_uri` | `None` | セッション永続化のための URI (例: `sqlite:///session.db`)。 |
| `--log_level` | `INFO` | ログの詳細度 (`DEBUG`, `INFO`, `WARNING` 等)。 |

### クライアント切断時の挙動
このモードは **Uvicorn** を使用しているため、標準的な ASGI の切断処理を継承します：
1.  **検知**: Uvicorn が TCP ソケットの切断を検知します。
2.  **伝播**: FastAPI/Starlette アプリケーションに `http.disconnect` イベントを送信します。
3.  **キャンセル**: Starlette がリクエスト処理タスクをキャンセルします (`asyncio.CancelledError`)。
4.  **結果**: ADK エージェントの実行は直ちに中断されます。部分的なストリーミングバッファは消失し、SessionMemory には保存されません。

## 2. 開発用インターフェースモード (`adk web`)

このコマンドは `api_server` と同じ API サーバーを起動しますが、開発やテストのための Web UI (Angular) も配信します。

- **コマンド**: `adk web [AGENTS_DIR] [OPTIONS]`
- **主なユースケース**: ブラウザ UI を使用したローカル開発および対話的テスト。
- **技術スタック**: `adk api_server` と同一 (Uvicorn + FastAPI)。
- **相違点**: 開発者用 UI の静的アセットを配信するルートが含まれます。

メモリおよび切断時の挙動は `adk api_server` と **完全に同一** です。

---

## 3. インタラクティブ CLI モード (`adk run`)

このコマンドは、ターミナル上でエージェントを対話的に実行します。

- **コマンド**: `adk run [AGENT_PATH]`
- **主なユースケース**: Web サーバーを介さない、エージェントロジックの迅速なローカルテスト。
- **技術スタック**:
    - **Web サーバーなし**: Uvicorn や FastAPI は使用しません。
    - **入出力**: 標準入力 (`stdin`) と標準出力 (`stdout`) を使用します。
    - **Runner**: Web モードと同じ基盤の `Runner` クラスを使用します。

### 中断時の挙動
切断すべき "HTTP クライアント" は存在しませんが、**KeyboardInterrupt (Ctrl+C)** が同様の役割を果たします：
1.  **中断**: Python プロセスが割り込みシグナルを受信します。
2.  **キャンセル**: 非同期ループがキャンセルされます。
3.  **結果**: Web モードと同様に、バッファリングを行っている `StreamingResponseAggregator` は `close()` を呼び出すことなく停止するため、部分的なレスポンスは生成されず、保存もされません。

## スタックとバッファ挙動のまとめ

| モード | コマンド | サーバースタック | SSE サポート | 切断/中断の処理 |
| :--- | :--- | :--- | :--- | :--- |
| **デーモン** | `adk api_server`| Uvicorn + FastAPI | あり (`/run_sse`) | `http.disconnect` -> タスクキャンセル -> バッファ消失 |
| **Web UI** | `adk web` | Uvicorn + FastAPI | あり (`/run_sse`) | `http.disconnect` -> タスクキャンセル -> バッファ消失 |
| **CLI** | `adk run` | Pure Python CLI | なし (テキスト出力) | `Ctrl+C` -> ループキャンセル -> バッファ消失 |
