# google-adk パッケージアップデート内容 (1.22.1 → 1.25.1)

## 概要

`google-adk` パッケージをバージョン `1.22.1` から `1.25.1` にアップデートした際の主要な機能面での変更点をまとめます。
（調査対象バージョン： `1.23.0`、`1.24.0`、`1.24.1`、`1.25.0`、`1.25.1`）

主な変更点は、以下の4つの領域に分類されます。

### 1. コア機能・アーキテクチャの強化
* **セッション管理の改善**
  * `adk api_server` での `--auto_create_session` オプションの追加。セッションが存在しない場合に自動生成される機能が追加されました。（※ 1.25.0 で追加）
  * トークン数に応じた自動コンパクション（圧縮）とイベント保持機能が追加され、メモリ管理が向上しました。（※ 1.25.0 で追加）
* **設定・デプロイの柔軟性向上**
  * Agent Engineへのデプロイ時に `GOOGLE_CLOUD_LOCATION` のパススルーがサポートされました。
  * CLIでの機能の有効/無効を制御する `--enable_features` と `--disable_features` フラグが追加されました。
* **最適化・実験のためのインターフェース**
  * エージェントのオプティマイザ（プロンプト最適化など）に向けたインターフェースとデモが追加されました。

### 2. Web UI と開発者体験（DX）の大幅な向上（特に1.24.0）
* **A2UI の統合とUI刷新**
  * Web UI（`adk-web`）が強化され、チャット機能内で直接 A2UI (Agent to UI) のネイティブコンポーネントをレンダリングできるようになりました。
  * 従来のイベントタブが廃止され、各メッセージ行をクリックして展開する直感的なデバッグビュー（Consolidated Event View）に改善されました。
* **開発者向け便利機能**
  * 関数呼び出し時のツールチップで引数や状態変化（レスポンス等）を即座に確認できるようになりました。
  * 矢印キーによるフルナビゲーションなど、キーボードに最適化されたUI操作が可能になりました。
  * 1.25.0 では、Webサーバーに APIのヘルスチェック用 `/health` およびバージョン確認用の `/version` エンドポイントが追加されています。

### 3. ツール (Tools) と モジュール連携の拡張
* **MCP（Model Context Protocol）の強化**
  * MCP ツールセットや OpenAPI ツールセットを含む各ツールセットで認証機能（`McpToolset` での動的ヘッダーや認証等）が導入されました。
  * MCPリソースへのアクセス・ロードを行うための `ロード機能 (load MCP resource tool)` やメソッドが新たに実装されました。
* **ツールの拡充と柔軟性の向上**
  * 新たに `SkillToolset` が追加されました。
  * `VertexAiSearchTool` が動的な構成パラメータ（Dynamic config）をサポートしました。
    * **[詳細]**: `VertexAiSearchTool` のサブクラスを作成し `_build_vertex_ai_search_config` メソッドをオーバーライドすることで、実行時のコンテキスト（セッションの状態など）に基づいて `filter`, `datastore`, `engine`, `max_results` 等のパラメータを動的に変更できるようになりました。これにより、ユーザー権限に応じたデータソースの切り替えや、会話の流れに応じた柔軟な検索フィルタリングが実現可能です。
  * Google API の認証において、外部から渡されたアクセストークンにも対応できるようになりました。

### 4. モデル設定（Models）、評価（Evals）、ライブモード
* **Models（推論周辺）**
  * Gemini LLM 実装に `base_url` オプションが追加されました。
  * `generate_content_config` に `thinking_config` （推論/思考プロセスの設定）が指定できるようになり、思考過程をモデル設定に組み込めるようになりました。
* **Evals（評価機能）**
  * `adk eval` コマンドでカスタムメトリクスが利用可能になり、テストの柔軟性が向上しました。評価結果を評価ダッシュボード等に連動させるレポート生成機能も追加されています。
* **Live モードとストリーミング**
  * 双方向ストリーミングのライブモードにおいて、ユーザー入力のセッションへの永続化が追加されました。
  * ツール呼び出しを別スレッドで実行する機能がサポートされ、ライブモードでのパフォーマンスと安定性が向上しています。
